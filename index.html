title:HTML5概述

  • HTML5历史

        HTML5发源于World Wide Web Consortium(简称W3C),是Web标准的一种.它是在2004年8月W3C关于Web应用程序以及复合文档(Compound Documents)的研讨会上诞生的,但是HTML5的的发展却没有W3C的参与,初始W3C并不是赞成HTML5而是比较推崇XHTML、XForms、SVG、以及SMIL.这是一件有意思的一件事,HTML5的发展可以说是站在W3C的竞争立场上进行发展,主要是一些浏览器厂商(Safari、Opera等)组成的WHATWG(Web Hypertext Applications Technology Working Group)社区以及HTML社区共同决定了HTML5.

      上图是WHATWG 主页的主页,任何需要帮助的HTML5的使用者都可以主页的论坛上的到帮助.WHATWG和一些浏览器开发者聚集在一起,共同起草规范,对于HTML5的讨论非常激烈,各种针锋相对的见解被提出来.

      在WHATWG致力更新HTML的同时,W3C的XHTML2工作组正在试图建立一个崭新的Web.这里可以简单提一下,期初的Web开发没有一个标准化,各个开发人员自成一派导致Web开发很难学习、阅读以及维护.W3C就以XML的严谨格式来进行了Web进行了规范标准的制定代表就是XHTML.

      2002年8月5日W3C发布XHTML 2.0的第一个工作草案,令人最吃惊的是,它与以往的版本不一样的是它不是向后兼容,而且进行对于XHTML语言标签的大量修改和更替.这与WHATWG希望实现兼容的做法不同.由于W3C的改变过于激进而没有考虑既有的网络环境的实现,缺乏浏览器厂商的支持,最终XHTML 2工程最终宣告死亡.

      在2006年,双方决定进行合作,来创建一个新版本的 HTML.2007年3月,W3C宣布成立新的HTML工作组成立,并邀请WHATWG所有的成员.之后共同起草新的规范,并融入HTML5.在W3C的章程计划中在2010年的第三个季度推出HTML5,对此WHATWG认为这么快是不可能.最终在2014年10月29日，万维网联盟宣布，经过接近8年的艰苦努力，该标准规范终于制定完成.HTML5将会取代1999年制定的HTML 4.01、XHTML 1.0标准，以期能在互联网应用迅速发展的时候，使网络标准达到符合当代的网络需求，为桌面和移动平台带来无缝衔接的丰富内容.

  • HTML5优势

网络标准

HTML5本身是由W3C推荐出来的，它的开发是通过谷歌、苹果，诺基亚、中国移动等几百家公司一起酝酿的技术，这个技术最大的好处在于它是一个公开的技术。换句话说，每一个公开的标准都可以根据W3C的资料库找寻根源。另一方面，W3C通过的HTML5标准也就意味着每一个浏览器或每一个平台都会去实现。

多设备跨平台

用HTML5的优点主要在于，这个技术可以进行跨平台的使用。比如你开发了一款HTML5的游戏，你可以很轻易地移植到UC的开放平台、Opera的游戏中心、Facebook应用平台，甚至可以通过封装的技术发放到App Store或Google Play上，所以它的跨平台性非常强大，这也是大多数人对HTML5有兴趣的主要原因。

自适应网页设计

很早就有人设想，能不能“一次设计，普遍适用”，让同一张网页自动适应不同大小的屏幕，根据屏幕宽度，自动调整布局（layout）。
2010年，Ethan Marcotte提出了”自适应网页设计“这个名词，指可以自动识别屏幕宽度、并做出相应调整的网页设计。
这就解决了传统的一种局面——网站为不同的设备提供不同的网页，比如专门提供一个mobile版本，或者iPhone / iPad版本。这样做固然保证了效果，但是比较麻烦，同时要维护好几个版本，而且如果一个网站有多个portal（入口），会大大增加架构设计的复杂度。

即时更新

游戏客户端每次都要更新，很麻烦。可是更新HTML5游戏就好像更新页面一样，是马上的、即时的更新。

总结概括HTML5有以下优点：
1、提高可用性和改进用户的友好体验；
2、有几个新的标签，这将有助于开发人员定义重要的内容；
3、可以给站点带来更多的多媒体元素(视频和音频)；
4、可以很好的替代FLASH和Silverlight；
5、当涉及到网站的抓取和索引的时候，对于SEO很友好；
6、将被大量应用于移动应用程序和游戏；
7、可移植性好。

  • HTML5的发展

      从如今层出不穷的移动应用就知道，在这个智能手机和将平板电脑大爆炸的时代，移动优先已成趋势，不管是开发什么，都以移动为主.据统计2013年全球将有10亿手机浏览器支持HTML5，同时HTML Web开发者数量将达到200万。有浏览器的地方,就需要HTML5!毫无疑问，HTML5将成为未来5-10年内，移动互联网领域的主宰者.如当下的微信平台就将HTML5点爆.

      随着技术的发展,HTML5不在仅仅局限于Web的开发,现在流应用的概念也是越来越热,也就是说基于HTML5的App一点也不必原生的App差,而且可跨平台的特性也是其它语言无法比拟的.

  • HTML5简看

      了解完HTML5的历史以及优势和发展,我们现在来简单看下HTML5的元素及属性本身.我们在网页上输出一个Hello World!

每一段HTML5代码都会以一句<!DOCTYPE html>作为开端.如果你有使用HTML之前的经验,你会发现代码被简化了!对于HTML5主要由head和body中组成,我们在body中进行内容的布局,在head中进行风格样式的控制.

title: JavaScript 预热

JavaScript简介

大概在1992年,一家称作 Nombas的公司开发了一种叫做C减减(C-minus-minus,简称Cmm)的嵌入式脚本语言.Cmm背后的理念很简单:一个足够强大可以替代宏操作(macro)的脚本语言,同时保持与C(和C++)足够的相似性,以便开发人员能很快学会.Nombas最终把Cmm名字改为ScriptEase.

在Netscape Navigator2.0即将正式发布前,Netscape 将其更名为 JavaScript,目的是为了利用Java这个因特网时髦词汇.Netscape的赌注最终得到回报,JavaScript从此变成了因特网的必备组件.

JS中的基本数据类型

JS中的变量是弱类型的,这与Java和C不同,变量的类型无特定的类型,定义变量时只用var运算符,就可以将变量初始化任意类型值.

例子:

  • Undefined 类型

Undefined类型只有一个值,即undefined.当声明的变量未初始化时,该变量的默认值是undefined.

  • Null 类型

Null类型只有一个专属值null.值undefined实际上是从值null派生来的,因此它们被定义为相等的.

  • Boolean 类型

Boolean有两个值true和false.

  • Number 类型

这种类型是比较特殊的数据类型.这种类型既可以表示32位的整数,还可以表示64位的浮点数.直接输入的任何数字都被看做Number类型.

另外整数也可以被表示为八进制或者十六进制的数.八进制数的首字母必须为0,其后的数字可以是任何八进制数字(0-7).

对于十六进制,必须以0x开头,然后是任意的十六进制数字(0-9和A-F).字母可以是大写也可以是小写.

  • String 类型

String类型一直在编程语言中都是比较有意思的类型,在JS中也是,它是唯一没有固定大小的原始类型.可以用字符串存储0或者更多的Unicode字符.
另外字符串中每一个字符都有特定的位置.首字符从位置0开始,往后依次类推.

本文只是JS的预热,之后会持续更新更加详细的JS知识.
